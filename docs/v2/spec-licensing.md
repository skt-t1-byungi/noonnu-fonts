## 목표

V2의 핵심 리스크는 “폰트 파일을 패키지에 포함해 재배포”하는 행위다. 따라서 라이선스는 “대충 요약”이 아니라 **증거 기반 정책**으로 관리한다.

이 문서는 다음을 정의한다:

- 어떤 경우에 배포 가능한지(정책)
- 어떤 증거가 필요한지(증거 요구사항)
- 애매할 때 어떻게 처리하는지(검토/차단 플로우)

## 기본 원칙(권장)

- **원칙 1**: “재배포 가능”이 명확하지 않으면 배포하지 않는다(`needs_review`).
- **원칙 2**: 모든 `approved` 엔트리는 라이선스 원문 또는 약관 페이지의 스냅샷(증거)을 가진다.
- **원칙 3**: LLM은 “필드 추출/요약”만 가능하며, **최종 승인**은 사람(또는 강한 규칙)이 한다.
- **원칙 4**: 라이선스가 변경될 수 있으므로, 승인 시점의 증거를 보관한다.

## 상태 정의

- `blocked`
  - 명시적으로 재배포 금지, 또는 자동 수집/다운로드 자체가 명확히 금지인 경우
- `needs_review`
  - 증거 부족/모호/충돌, 또는 품질 검증 실패로 보류
- `approved`
  - 재배포 가능이 명확 + 증거 확보 + 필수 메타 충족

## 증거 요구사항(최소)

`approved`의 최소 조건(권장):

- 라이선스/이용조건을 설명하는 URL 최소 1개
- 해당 페이지의 스냅샷(HTML/PDF/TXT)과 해시(`sha256`)
- “재배포 허용”을 뒷받침하는 문구가 원문에서 확인 가능해야 함

추가 권장 증거:

- 다운로드 페이지 스냅샷(다운로드 조건이 별도로 있을 수 있음)
- 배포 주체(기관/기업) 공식 도메인 확인

## 정책 룰(초기 규칙 예시)

다음 중 하나라도 만족하면 기본적으로 `needs_review`:

- “재배포(redistribution)”에 대한 언급이 없고, 단지 “사용 가능”만 있는 경우
- “개인/비상업적만 허용” 등의 제한이 있는 경우(상업적 사용 목표라면)
- 폰트 파일의 제공 경로가 로그인/약관 동의/유료 결제 등과 결합되어 있는 경우

다음은 즉시 `blocked` 후보:

- “재배포 금지”, “파일 수정/변형 금지”가 명시되었고 V2의 배포 형태와 충돌하는 경우
- 자동화된 수집을 금지하는 조항이 명확하고, 우회가 필요해지는 경우

## 리뷰 프로세스(권장)

1) Collector가 후보 생성 → `needs_review`
2) Evidence Harvester가 원문 저장
3) License Classifier가 정책 필드 채움(자동)
4) 사람이 원문/증거 확인 후:
   - `approved`로 승격 또는
   - `blocked`로 차단 + 사유 기록
5) `approved`만 패키징/퍼블리시 가능

## 패키지에 포함해야 하는 라이선스 정보(권장)

패키지 디렉터리에는 최소 아래 중 하나를 포함:

- `LICENSE.md`: 원문 전문 또는 원문 링크 + 요약 + 핵심 조건(상업적/재배포/출처표기)
- `EVIDENCE.md`: 증거 URL, 스냅샷 경로, 해시, 캡처 시각
