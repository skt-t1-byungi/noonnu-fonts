## V2 목표

이 저장소의 V2는 “특정 사이트(예: 눈누) 크롤링”이 아니라, **독자적인 한글 폰트 레지스트리**를 구축하고, **라이선스/출처/증거**를 함께 관리하면서, 웹/앱에서 재사용 가능한 형태로 **고품질 패키지로 퍼블리싱**하는 것을 목표로 한다.

핵심 요구사항:

- **고품질 퍼블리싱**: woff2 중심(필요 시 woff/ttf 포함), CSS 생성, 예제 이미지/샘플, 메타데이터, 무결성 해시 제공
- **라이선스 안전성**: “재배포 가능”을 증거 기반으로 관리하고, 불명확한 항목은 자동 제외 + 검토 대기
- **독자적 수집/갱신**: 검색/수집 소스를 플러그인으로 확장(웹 탐색, GitHub 릴리즈, 기관 배포 페이지 등)
- **재현 가능**: 동일 입력(레지스트리/아티팩트)에서 동일 산출물이 나오도록 파이프라인을 결정론적으로 설계

## V2 문서

- `docs/v2/architecture.md`: 전체 아키텍처(컴포넌트, 데이터 흐름, 경계)
- `docs/v2/spec-registry.md`: 레지스트리 스키마(소스 오브 트루스)와 파일 레이아웃
- `docs/v2/spec-pipeline.md`: 파이프라인 단계(수집 → 증거 수집 → 검증 → 패키징 → 퍼블리싱)
- `docs/v2/spec-licensing.md`: 라이선스 정책/증거 요구사항/검토 프로세스
- `docs/v2/roadmap.md`: 단계별 전략 및 마이그레이션/운영 계획
